# ALB with path based routing

Is lab mein hum Application Load Balancer ke saath path based routing implement karenge. Pichli lab mein humne host based routing implement ki thi. Dono lab setup steps, kaafi same hain but routin kar time chize different hongi.

To hum ye assume karenge ki humne pichli lab ka use karke kuch setup kar liya hai, but main path based routing is lab mein start karenge.

Path-Based Routing tab use hoti hai jab aapka Domain Name same rehta hai, lekin URL ke piche ka Path (Folder) badalne par traffic alag-alag backend servers par jata hai.

<br>
<br>

### Path-Based Routing Kya Hai?

Maan lijiye aapki ek website hai: ```mycompany.com```.
- Agar koi user normal site khole ```mycompany.com```, toh traffic aapke main web servers par jaye.
- Agar koi user video dekhne ke liye ```mycompany.com/video``` par jaye, toh ALB request ko pakad kar Video Processing Servers ke target group par forward kar de.
- Agar koi user image dekhne ke liye ```mycompany.com/image``` par jaye, toh traffic Image Optimization Servers par jaye.

<br>

### Pichli Lab Ke According Kya-Kya Badlega?

Kyunki aapka Custom VPC network, Gateways, Route Tables aur Security Groups pehle se setup hain, hume poori lab scratch se karne ki zaroori nahi hai. Hume pichli lab mein sirf 3 jagah changes karne honge:

**Change 1: Web Servers Ke Andar Ka Folder Structure (Step 5 Mein Change)**:

Nginx default roop se ```/usr/share/nginx/html/index.html``` file ko read karta hai. Path-based routing ke liye Nginx ke andar un paths ke naam ka folder hona mandatory hai, varna ```404 Not Found``` ka error aayega.

Isliye hum User Data mein folders banayenge:

**Blog-Server (Private 1A) ka naya User Data**:
```
#!/bin/bash
sudo dnf update -y
sudo dnf install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
# Pehle 'blog' naam ka folder banayenge
sudo mkdir -p /usr/share/nginx/html/blog
# Us folder ke andar index file dalenge
echo "<h1>Welcome to /blog Path Application</h1>" | sudo tee /usr/share/nginx/html/blog/index.html
```

**App-Server (Private 1B) ka naya User Data**:
```
#!/bin/bash
sudo dnf update -y
sudo dnf install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
# Pehle 'app' naam ka folder banayenge
sudo mkdir -p /usr/share/nginx/html/app
# Us folder ke andar index file dalenge
echo "<h1>Welcome to /app Path Dashboard</h1>" | sudo tee /usr/share/nginx/html/app/index.html
```

(Note: Target Groups blog-tg aur app-tg bilkul pichli lab ki tarah hi same rahenge aur unme yahi servers mapped rahenge).

<br>

**Change 2: Target Groups Ki Health Check Settings (Step 6 Mein Change)**:

Pichli lab mein ALB check kar raha tha ki port 80 par ```/ ```(main page) chal raha hai ya nahi. Ab hamare servers par main page khali hai, asli code ```/blog``` aur ```/app``` ke andar hai. Agar humne health check path nahi badla, toh ALB servers ko Unhealthy ghoshit kar dega.
- ```blog-tg``` ki settings mein: Health check section mein jaakar Path ko ```/``` se badal kar ```/blog/index.html``` kar dein.
- ```app-tg``` ki settings mein: Health check section mein jaakar Path ko ```/``` se badal kar ```/app/index.html``` kar dein.

<br>

**Change 3: ALB Ke Rules (Step 8 Mein Change)**:

Pichli lab mein humne listener rules mein **Host Header** condition chuni thi, is baar hum use delete karke **Path condition** lagayenge.
- ```routing-alb``` ke Listeners and routing tab mein jaakar rules ko edit karein.
- Purane rules ko mita kar naye rules is tarah banayein:
  
  - **Rule 1 (Blog Path Rule)**:
    - **Condition**: Dropdown se chunein Path. Value mein type karein: ```/blog*``` (Asterisk ```*``` lagana zaroori hai taaki ```/blog``` ke aage ka saara traffic isme cover ho jaye).
    - **Action**: Forward to ```blog-tg```.
   
  - **Rule 2 (App Path Rule)**:
    - **Condition**: Dropdown se chunein Path. Value mein type karein: ```/app*```.
    - **Action**: Forward to ```app-tg```.

### Verification (Ab Ise Test Kaise Karenge?)

Is baar aapko laptop ki hosts file mein koi badlaav nahi karna padega, aap direct Load Balancer ke default lambe DNS URL se hi ise test kar sakte hain:
- Load Balancer ka DNS Name copy karein (Jaise: ```://amazonaws.com```).
- Browser mein naya tab kholein aur URL ke aage manually path lagayein:

  - **Test A**: ```http://amazonaws.com/blog```.
  - **Output**: ```Welcome to /blog Path Application```.
 
  - **Test B**: ```http://amazonaws.com/images```.
  - **Output**: ```Welcome to /app Path Dashboard```.
 
Aapka single domain/URL ab piche se alag-alag paths ke basis par traffic divide kar raha hai.
