# LAB: Application Load Balancer with custom domain name

Is LAB mein hum ye dekhenge ki agar humare pass khud ka domain name hai, jisko hum apni website pe use karna chate hain. Lekin traffic Application Load Balancer hi servers par distribute kare.

Pichle lab mein humne ek Lab kari thi: Jisme ALB 2 ec2 instances jo ki private subnet mein hote hain us par traffic distribute karta hai. 

To aap assume kariye ki pichli lab ka use karke humne already ALB aur EC2 instances ka complete setup kiya hua hai aur ALB ke khud ke DNS se hum servers pe deployed web application ko access kar pa rahe hain.

Ab aage is lab ko use ke baad continue karenge.

<br>
<br>

## ALB with Custom Domain Name

Production setup mein jab hum website launch karte hain, toh hum kabhi bhi user ko Load Balancer ka lamba URL (jaise ```://amazonaws.com```) nahi dete. Hum apna ek saaf-suthra custom domain use karte hain, jaise ```myproductionwebsite.com```.

AWS mein domain name ko Load Balancer se jodne ke liye hum **Route 53** (AWS ki DNS Service) ka use karte hain.

Chaliye, hum apni purani custom VPC wali lab mein Custom Domain setup karne ke steps click-by-click add karte hain.

<br>
<br>

### Step 10: Domain Name Register Ya Manage Karna (Route 53 Setup)

Maan lijiye aapke paas pehle se ek domain hai (chahe aapne use GoDaddy, Namecheap se kharida ho ya AWS Route 53 se). Hume AWS mein uski ek **Hosted Zone** banani padegi.

- AWS Console ke search bar mein **Route 53** type karke dashboard par jayein.
- Left menu se **Hosted zones** par click karne ke baad, **Create hosted zone** button par click karein.
- **Domain name**: Apna domain name dalein, jaise: ```myproductionwebsite.com```.
- **Type**: ```Public hosted zone``` select karein (kyunki ise internet se access hona hai).
- Sabse neeche jaakar **Create hosted zone** par click kar dein.

_(Note: Agar aapka domain GoDaddy ya kisi aur provider par hai, toh Route 53 aapko 4 Name Servers [NS Records] dega. Aapko un chaaro lines ko copy karke apne GoDaddy account ke DNS Settings mein jaakar 'Custom Name Servers' mein paste karna hoga. Isse aapka domain AWS se link ho jata hai)._

<br>
<br>

### Step 11: Domain Ko Load Balancer Se Connect Karna (Alias Record)

Ab hum ek aisa pointer (Record) banayenge jo domain par aane wale traffic ko Load Balancer ki taraf redirect karega.

- Apni bani hui Hosted Zone (```myproductionwebsite.com```) ke andar jayein.
- Right side mein orange color ke **Create record** button par click karein.
- **Routing policy**: ```Simple routing``` select rehne dein aur Next karein.
- **Configure records** page par **Define simple record** par click karein:
  - **Record name**: Agar aap chahte hain ki website ```://myproductionwebsite.com``` par khule, toh box mein ```www``` likhein. Agar aap main domain ```myproductionwebsite.com``` chahte hain, toh ise khali chhod dein.
  - **Value/Route traffic to**: Dropdown se select karein: **Alias to Application and Classic Load Balancer**.
  - **Choose Region**: Woh region select karein jahan aapne apni Load Balancer lab banayi hai (jaise ```ap-south-1``` Mumbai ya ```us-east-1``` N. Virginia).
  - **Choose load balancer**: Dropdown par click karte hi aapko apni purani lab ka Load Balancer ```scratch-web-alb``` automatic dikh jayega, use select kar lein.
 
- **Define simple record** button par click karein, aur fir **Create records** par click kar dein.

<br>
<br>

### Step 12: Custom Domain Par Security Lagana (SSL/TLS Certificate - ACM)

Kyunki pichli baar hamari website browser par sirf ```http://``` lagane par khul rahi thi, production standard ke mutabik hume use ```https://``` (Secure lock icon) banana hoga. Iske liye AWS hume free certificate deta hai ACM (AWS Certificate Manager) ke zariye.

<br>

**Part A: Certificate Request Karna**:
- Search bar mein **Certificate Manager** (ACM) type karke dashboard par jayein.
- Top right mein **Request** button par click karein, fir **Request a public certificate** select karke Next karein.
- **Domain name**: Type karein ```myproductionwebsite.com``` aur ek aur line add karke likhein ```*.myproductionwebsite.com``` (taaki www ya koi bhi sub-domain automatic secure ho jaye).
- Validation method mein **DNS validation** chunein aur Request par click kar dein.
- Ab aapko screen par status _Pending validation_ dikhega. Us certificate ke upar click karein.
- Andar aapko ek button dikhega **Create records in Route 53**. Us par click kar dein. AWS automatic aapke Route 53 mein jaakar validation entries dal dega. 5 minute mein aapka certificate status green color mein **Issued** ho jayega.

<br>

**Part B: Load Balancer Par HTTPS Listener Add Karna**:

Ab hume Load Balancer ko sikhana hoga ki woh Port 443 (https) ka traffic bhi accept kare.
- **EC2 Dashboard -> Load Balancers** par jayein aur apne ```scratch-web-alb``` par click karein.
- **Listeners and routing** tab mein jaakar **Add listener** button par click karein.
- **Protocol** : Port mein select karein: ```HTTPS``` (Port automatic 443 ho jayega).
- **Default actions**: Select karein Forward to target groups aur apna purana backend group ```backend``` select kar lein.
- **Secure listener settings**: Scroll karke neeche jayein. From ACM select karein aur dropdown se apna abhi-abhi banaya naya SSL Certificate select kar lein.
- **Save** par click kar dein.

<br>
<br>

### Final Custom Domain Request Flow (Traffic Kaise Ghumega?)

Ab jab poori duniya se koi user aapka custom domain hit karega:
- User ne browser mein type kiya: ```https://myproductionwebsite.com```.
- **Route 53 (DNS Check)**: Browser sabse pehle AWS Route 53 ke paas jayega aur poochega ki is naam ka IP kya hai? Route 53 dekhega ki ispar **Alias** Record laga hai jo ```scratch-web-alb``` se juda hai. Woh request ko Load Balancer ki taraf bhej dega.
- **ALB HTTPS Listener Check**: Request Load Balancer ke **Port 443 (HTTPS)** par land karegi. Load Balancer apna SSL Certificate browser ko dikhayega, jisse browser par Green Lock (Secure) icon ban jayega.
- **Backend Delivery**: Load Balancer traffic ko decrypt karega aur normal HTTP Port 80 ke zariye **Private Subnet** mein chhupkar baithe ```Web-Server-A``` ya ```Web-Server-B``` ko request transfer kar dega.
