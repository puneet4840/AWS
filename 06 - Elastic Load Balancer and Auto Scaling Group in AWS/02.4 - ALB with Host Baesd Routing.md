# LAB 4: ALB with Host Based Routing

Is lab mein hum ALB ko host-based routing ke saath implement karenge. Setup same rahega, 2 private subnets mein ec2 instances rahenge aur alb host-based routing ke through ec2 instances par traffic distribute karega.

Application Load Balancer (ALB) ka sabse powerful feature hai Host-Based Routing (jise Layer 7 Routing bhi kehte hain). Iska matlab hai ki aap ek single Load Balancer aur ek single IP/DNS Name ka use karke alag-alag websites ya microservices ko chala sakte hain.

Host based routing ka matlab hai ki host ke basis par traffic ko particular ec2 instance par bhejna, jaise ```app.mycompany.com``` ko app webserver par bhjena. Aur ```blog.mycompany.com``` ko blog server par bhejna.

<br>
<br>

### Host-Based Routing Kya Hai?

Maan lijiye aapki company ke paas do alag-alag web applications hain:
- Ek hai blog site: ```http://blog.mycompany.com```.
- Ek hai main app: ```http://app.mycompany.com```.

Pehle zamane mein in dono ke liye alag-alag Load Balancer aur alag-alag IP kharidne padte the, jisse cost bohot badh jati thi.

**Solution**: Application Load Balancer internet se aane wali HTTP request ka Host Header (yaani domain name jo user ne type kiya hai) read kar sakta hai.
- Agar user ne browser mein ```http://blog.mycompany.com``` dala, toh ALB request ko pakad kar Blog wale Target Group (Servers) par bhej dega.
- Agar usi user ne ```http://mycompany.com``` dala, toh vahi same ALB request ko App wale Target Group par route kar dega.

<br>
<br>

### Lab Scenario Overview

Hum ek complete Production-ready infrastructure shuru se (scratch se) custom VPC ke saath banayenge.
- **1 Custom VPC** (```routing-vpc```).
- **2 Public Subnets** (ALB ke liye across 2 Zones).
- **2 Private Subnets** (Backend servers ke liye across 2 Zones).
- **Target Group 1** (```blog-tg```): Isme ek server chalega jo ```Welcome to Blog Application dikhayega```.
- **Target Group 2** (app-tg): Isme dusra server chalega jo ```Welcome to Main Dashboard App dikhayega```.
- **1 Application Load Balancer**: Jo host headers ke mutabik traffic divide karega.


<br>
<br>

### Step 1: Custom VPC Network Khada Karna (Scratch Se)

- AWS Console mein **VPC Dashboard** par jayein. Left menu se **Your VPCs **select karein aur **Create VPC** par click karein.
- Select karein ```VPC only```:
  - Name tag: ```routing-vpc```.
  - IPv4 CIDR block: ```192.168.0.0/16```. 
- Sabse neeche jaakar **Create VPC** par click kar dein.

<br>
<br>

### Step 2: 4 Subnets Create Karna

- Left menu se **Subnets -> Create subnet** par click karein.
- **VPC ID**: Apna banaya naya ```routing-vpc``` select karein.
- **Add new subnet** button ka use karte hue neeche diye gaye charo subnets configure karein:
  - **Subnet 1** (Public 1A): Name: ```pub-subnet-1a``` | AZ: ```us-east-1a``` (ya aapka zone A) | CIDR: ```192.168.1.0/24```.
  - **Subnet 2** (Public 1B): Name: ```pub-subnet-1b``` | AZ: ```us-east-1b``` (ya aapka zone B) | CIDR: ```192.168.2.0/24```.
  - **Subnet 3** (Private 1A): Name: ```private-subnet-1a``` | AZ: ```us-east-1a``` | CIDR: ```192.168.3.0/24```.
  - **Subnet 4** (Private 1B): Name: ```private-subnet-1b``` | AZ: ```us-east-1b``` | CIDR: ```192.168.4.0/24```.
- **Create subnet** par click kar dein.
- Subnets list mein ```pub-subnet-1a``` par tick lagayein, **Actions -> Edit subnet settings** par jayein, **Enable auto-assign public IPv4** address ko tick karke Save karein. Same yahi step pub-1b ke liye bhi repeat karein.

<br>
<br>

### Step 3: Gateways Aur Route Tables Configuration

- Left menu se **Internet Gateways -> Create internet gateway**. Name dein: ```routing-igw``` aur Create karein.
- Is naye IGW ko select karke **Actions -> Attach to VPC** par click karein aur ```routing-vpc``` se link kar dein.
- Left menu se **NAT Gateways -> Create NAT gateway**. Name dein: ```routing-nat```. Subnet dropdown se strictly ```pub-subnet-1a``` select karein. **Allocate Elastic IP** par click karke Create kar dein.

**Route Tables Setup**:
- **Route Tables** -> **Create route table**. Name: ```pub-rt``` | VPC: ```routing-vpc``` | Create karein.
  - **Routes tab** -> **Edit routes**: Add route karke ```0.0.0.0/0``` select karein aur Target mein ```Internet Gateway``` (```routing-igw```) chunein. Save changes.
  - **Subnet associations tab** -> **Edit**: Apne dono public subnets (```pub-subnet-1a```, ```pub-subnet-1b```) ko check mark karke Save kar dein.
 
- Firse **Create route table**. Name: ```private-rt``` | VPC: ```routing-vpc``` | Create karein.
  - **Routes tab** -> **Edit routes**: Add route karke ```0.0.0.0/0``` select karein aur Target mein ```NAT Gateway``` (```routing-nat```) chunein. Save changes.
  - **Subnet associations tab** -> **Edit**: Apne dono private subnets (```private-subnet-1a```, ```private-subnet-1b```) ko check mark karke Save kar dein.
 
<br>
<br>

### Step 4: Security Groups Taiyar Karna

Left menu se **Security Groups** -> **Create security group**:
- Name: ```alb-public-sg``` | VPC: ```routing-vpc```.
- Inbound Rule: HTTP (Port 80) | Source: ```Anywhere-IPv4``` (```0.0.0.0/0```). Create karein.

Firse **Create security grou**p:
- Name: ```backend-server-sg``` | VPC: ```routing-vpc```.
- Inbound Rule: HTTP (Port 80) | Source: _Custom_ select karke search karein ```alb-public-sg```. Create karein.

<br>
<br>

### Step 5: Private Subnets Mein Alag-Alag Web Servers Launch Karna

Hum do alag servers banayenge taaki unpar alag-alag applications (Blog aur Dashboard App) dikh sakein.

**Server 1 (Blog App - Private Subnet 1A)**:
- EC2 Dashboard -> Instances -> Launch instances.
- Name: ```Blog-Server``` | AMI: ```Amazon Linux 2023```.
- Network Settings ke aage **Edit** par click karein:
  - VPC: ```routing-vpc``` | Subnet: ```private-subnet-1a```.
  - Security Group: Existing chunein aur ```backend-server-sg``` select karein.
 
- **Advanced Details** -> **User data** box mein yeh script dalein:
```
#!/bin/bash
sudo dnf update -y
sudo dnf install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
echo "<h1>Welcome to Blog Application</h1>" | sudo tee /usr/share/nginx/html/index.html
```
- Launch instance par click kar dein.

<br>

**Server 2 (Main App - Private Subnet 1B)**:
- Firse **Launch instances** par click karein.
- Name: ```App-Server``` | AMI: ```Amazon Linux 2023```.
- Network Settings -> **Edit**:
  - VPC: ```routing-vpc``` | Subnet: ```private-subnet-1b```.
  - Security Group: ```backend-server-sg```.
 
- **Advanced Details** -> **User data** box mein yeh script dalein:
```
#!/bin/bash
sudo dnf update -y
sudo dnf install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
echo "<h1>Welcome to Main Dashboard App</h1>" | sudo tee /usr/share/nginx/html/index.html
```
- Launch instance par click kar dein.

<br>

### Step 6: Do Alag-Alag Target Groups Banana

ALB ko host header ke hisab se do alag targets (groups) par traffic forward karna hai, isliye hum do target groups banayenge.

Left menu se **Target Groups** -> **Create target group**.
- Type: Instances | Name: ```blog-tg``` | VPC: ```routing-vpc``` -> Next karein.
- Target list mein sirf ```Blog-Server``` par tick lagayein, _Include as pending below_ par click karein aur Target Group create kar dein.

Firse **Create target group** par click karein:
- Type: Instances | Name: ```app-tg``` | VPC: ```routing-vpc``` -> Next karein.
- Target list mein is baar sirf ```App-Server``` par tick lagayein, _Include as pending below_ karein aur ise bhi create kar dein.

<br>
<br>

### Step 7: Application Load Balancer (ALB) Create Karna

- Left menu se **Load Balancers** -> **Create load balancer** -> **Application Load Balancer** ke neeche Create par click karein.
- Name: ```routing-alb``` | Scheme: ```Internet-facing```.
- Network Mapping mein VPC chunein ```routing-vpc```. Mappings mein:
  - Zone 1a ke saamne dropdown se chunein ```pub-subnet-1a```.
  - Zone 1b ke saamne dropdown se chunein ```pub-subnet-1b```.
- Security Group: Default hata kar ```alb-public-sg``` chunein.
- Listeners and routing: HTTP:80 ke aage forward to mein kisi bhi ek group ko default select kar lein (jaise ```app-tg```). Abhi host rules hum agle step mein add karenge.
- **Create load balancer** par click kar dein.
