# LAB: Implementing Application L with 2 EC2s in same region with different subnets.

**Scenario**: Is LAB mein hum Application Load Balancer ke thorugh traffic ko 2 alag-alag private subnets mein chal rahe EC2 instances par distribute karenge.

Production environment mein hamesha web servers ko Private Subnets mein rakha jata hai taaki internet se koi unhe direct hack na kar sake. Public traffic sirf Load Balancer (ALB) tak aata hai jo ki Public Subnets mein hota hai, aur ALB piche se private servers par traffic bhejta hai.

<br>

### Architecture Overview

- **1 Custom VPC**: ```Load-Balancer-VPC```.
- **2 Public Subnets**: ALB ke liye (Zone A aur Zone B mein). Ye ALB ki high availability ke liye 2 public subnets hain.
- **2 Private Subnets**: EC2 Web Servers ke liye (Zone A aur Zone B mein). Dono EC2 alag-alag subnets mein deploy honge.
- **1 NAT Gateway**: Private servers ko internet se patch updates download karne dene ke liye (Optional par production standard).
- **1 Internet Gateway**: VPC ko internet se connect karne ke liye.
- **1 Application Load Balancer**: Public subnets mein.
- **2 EC2 Instances**: Private subnets mein.
