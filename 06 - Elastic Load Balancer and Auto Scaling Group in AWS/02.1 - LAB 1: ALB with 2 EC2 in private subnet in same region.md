# LAB: Implementing Application Load Balancer with 2 EC2s in same region with private subnets.

**Scenario**: Is LAB mein hum Application Load Balancer ke thorugh traffic ko 2 alag-alag private subnets mein chal rahe EC2 instances par distribute karenge.

Production environment mein hamesha web servers ko Private Subnets mein rakha jata hai taaki internet se koi unhe direct hack na kar sake. Public traffic sirf Load Balancer (ALB) tak aata hai jo ki Public Subnets mein hota hai, aur ALB piche se private servers par traffic bhejta hai.

<br>

### Architecture Overview

- **1 Custom VPC**: ```Load-Balancer-VPC```.
- **2 Public Subnets**: ALB ke liye (Zone A aur Zone B mein). Ye ALB ki high availability ke liye 2 public subnets hain.
- **2 Private Subnets**: EC2 Web Servers ke liye (Zone A aur Zone B mein). Dono EC2 alag-alag subnets mein deploy honge.
- **1 NAT Gateway**: Private servers ko internet se patch updates download karne dene ke liye (Optional par production standard).
- **1 Internet Gateway**: VPC ko internet se connect karne ke liye.
- **1 Application Load Balancer**: Public subnets mein.
- **2 EC2 Instances**: Private subnets mein.

### Step 1: Custom VPC Banana 

Sabse pehle hum ek VPC create karenge.
- **AWS** Console ke search bar mein VPC type karke **VPC** Dashboard par jayein.
- Left menu se **Your VPCs** par click karein aur top right mein orange color ke **Create VPC** button par click karein.
- Is baar settings mein strictly ```VPC only``` select karein.
- Settings fill karein:
  - **Name tag**: ```Load-Balancer-VPC```.
  - **IPv4 CIDR block**: Manual select karke type karein: ```10.0.0.0/16```.
  - Sabse neeche jaakar **Create VPC** par click kar dein. Aapka khali network taiyar hai.
 
<br>
<br>

### Step 2: 4 Subnets Create Karna (2 Public + 2 Private)

Hum 2 alag-alag Availability Zones (Zone A aur Zone B) mein subnets banayenge taaki high availability bani rahe.
- Left menu se **Subnets** par click karein, fir top right mein **Create subnet** par click karein.
- **VPC ID**: Dropdown se apna banaya naya VPC ```Load-Balancer-VPC``` select karein.

Ab hum ek-ek karke 4 subnets add karenge (**Add new subnet** button par click karte hue):
- **Subnet 1 (Public Zone A)**:
  - Subnet name: ```public-subnet-1a```.
  - Availability Zone: ```us-east-1a``` (ya aapke region ka pehla zone).
  - IPv4 CIDR block: ```10.0.1.0/24```.
 
- **Subnet 2 (Public Zone B)**: (Neeche scroll karke Add new subnet par click karein:
  - Subnet name: ```public-subnet-1b```.
  - Availability Zone: ```us-east-1b``` (ya aapke region ka doosra zone).
  - IPv4 CIDR block: ```10.0.2.0/24```.
 
- **Subnet 3 (Private Zone A)**: (Neeche scroll karke Add new subnet par click karein):
  - Subnet name: ```private-subnet-1a```.
  - Availability Zone: ```us-east-1a```.
  - IPv4 CIDR block: ```10.0.3.0/24```.
 
- **Subnet 4 (Private Zone B)**: (Neeche scroll karke Add new subnet par click karein):
  - Subnet name: ```private-subnet-1b```.
  - Availability Zone: ```us-east-1b```.
  - IPv4 CIDR block: ```10.0.4.0/24```.
 
- Sabse neeche jaakar **Create subnet** par click kar dein. Saare subnets ban gaye hain.

**Public Subnets Ka Automatic IP Settings On Karna**:

Kyunki humne inhe 'Public' bola hai, hume ek setting on karni hogi taaki inme launch hone wali cheezon ko public IP mile:
- Subnets list mein ```public-subnet-1a``` par tick lagayein.
- **Actions -> Edit subnet settings** par click karein.
- **Auto-assign IP settings** ke andar **Enable auto-assign public IPv4 address** par check mark lagayein aur Save kar dein.
- Same yahi step ```public-subnet-1b``` ke liye bhi repeat karein.

<br>
<br>

### Step 3: Internet Gateway (IGW) Banana Aur Attach Karna

Internet Gateway ke bina aapka VPC bahar ki duniya se (Internet) connect nahi ho sakta.
- Left menu se **Internet Gateways** par click karein -> **Create internet gateway** par click karein.
- Name tag dalein: ```LB-igw``` aur **Create** kar dein.
- Gateway banne ke baad woh ```Detached``` status mein hoga. Top right mein **Actions -> Attach to VPC** par click karein.
- Dropdown se apna ```Load-Balancer-VPC``` chunein aur **Attach internet gateway** par click kar dein.

<br>
<br>

### Step 4: NAT Gateway (NGW) Setup Karna (Private Servers Ke Liye)

Private subnets ke andar chal rahe servers ko Nginx package download karne ke liye internet chahiye, par secure tarike se. Uske liye NAT Gateway zaroori hai.

- Left menu se **NAT Gateways** par click karein -> **Create NAT gateway** par click karein.
- Name dalein: ```lb-nat-gw```.
- **Subnet**: Dhyan se dropdown se apna ```public-subnet-1a``` select karein. (NAT Gateway hamesha public subnet mein baithta hai).
- **Elastic IP allocation ID**: Saamne bane **Allocate Elastic IP** button par click karein, AWS khud ek static IP generate karke attach kar dega.
- **Create NAT gateway** par click kar dein. (Ise active hone mein 1-2 minute lagte hain, tab tak hum agla step karte hain).

<br>
<br>

### Step 5: Custom Route Tables Banana Aur Subnets Link Karna

Abhi tak saare subnets aapas mein jude hain par unhe nahi pata ki internet par kaise jana hai. Hum 2 Route Tables banayenge: Ek Public aur Ek Private.

**Part A: Public Route Table**:
- Left menu se **Route Tables** par click karein -> **Create route table**.
- Name: ```public-subnet-rt``` | VPC: ```Load-Balancer-VPC``` | **Create** kar dein.
- Nayi bani route table ko select karein aur neeche tabs mein se **Routes** tab par click karke **Edit routes** par click karein.
- **Add route** par click karein:
  - Destination: ```0.0.0.0/0``` (Matlab saara internet traffic).
  - Target: Dropdown se ```Internet Gateway``` chunein aur apne ```LB-igw``` ko select kar lein. **Save changes** karein.
- Ab isi table ke andar **Subnet associations** tab par jayein -> **Edit subnet association**s par click karein.
- Apne dono public subnets (```public-subnet-1a``` aur ```public-subnet-1b```) ke saamne check mark lagayein aur **Save associations** kar dein.

<br>

**Part B: Private Route Table**:
- Firse **Create route table** par click karein.
- Name: ```private-subnet-rt``` | VPC: ```Load-Balancer-vpc``` | **Create** kar dein.
- **Routes** tab -> **Edit routes** -> **Add route**:
  - Destination: ```0.0.0.0/0```.
  - Target: Is baar dropdown se ```NAT Gateway``` chunein aur apne ```lb-nat-gw``` ko select karein. **Save changes** karein.
- **Subnet associations** tab -> **Edit subnet associations**:
  - Apne dono private subnets (```private-subnet-1a``` aur ```private-subnet-1b```) ko select karke **Save associations** kar dein.
 
<br>
<br>

### Step 6: Security Groups Configuration

Network taiyar hai! Ab security lagate hain.
- Left menu se **Security Groups** -> **Create security group**.
- Name: ```alb-sg``` | VPC: ```Load-Balancer-vpc```.
- **Inbound rules -> Add rule**: Type: ```HTTP``` | Source: Anywhere-IPv4 (```0.0.0.0/0```). **Create** kar dein.

Uper ```alb-sg``` security group ka matlab hai ki jo bhi traffic internet se load balancer par aayega usko load balancer par allow karo.
  
- Firse **Create security group** par click karein.
- Name: ```web-server-sg``` | VPC: ```Load-Balancer-vpc```.
- **Inbound rules -> Add rule**: Type: ```HTTP``` | Source: _Custom_ select karke search karein ```alb-sg```. Create kar dein.

Is ```web-server-sg``` security group ka matlab hai ki jo bhi traffic internet se load balancer par aaya hai, usko load balancer ke security group se web server tak aane ke liye allow karo. Matlab sirf load balancer ka traffic hi ec2 instances tak aa paye baaki koi traffic ec2 instance tak naa aye aur block ho jaye.

<br>
<br>

### Step 7: Private Subnets Mein EC2 Web Servers Launch Karna

Ab hum private subnets mein ec2 instances ko launch karenge.
- **EC2 Dashboard** -> **Instances** -> **Launch instances**.
- Name: ```asia-webserver``` | AMI: ```Amazon Linux 2023``` | Key Pair: ```Proceed without a key pair```.
- Network Settings ke aage **Edit** par click karein:
  - VPC: ```Load-Balancer-vpc```.
  - Subnet: ```private-subnet-1a``` (Zone A ka private subnet).
  - Firewall: _Select existing_ chunein aur ```web-server-sg``` select karein.
- **Advanced Details** -> Sabse end mein User data box mein yeh script dalein, Ye user data script ec2 instance mein nginx server install karke uspe web-page run kar degi.
```
#!/bin/bash
sudo dnf update -y
sudo dnf install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
echo "<h1>Hello from Asia Webserver</h1>" | sudo tee /usr/share/nginx/html/index.html
```
- **Launch instance** kar dein.
- **Yahi se doosra instance launch karein**: Name: ```europe-webserver```, Subnet mein ```private-subnet-1b``` select karein, security group ```web-server-sg``` chunein, aur User Data script mein ```Hello from Europe Webserver``` likh kar launch kar dein.

<br>

### Step 8: Target Group Aur Application Load Balancer Setup

Ab hum Application Load Balancer ke liye backend setup karenge usme ec2 instances register karenge.
- **Target Groups** par jayein -> **Create target group**.
- Type: ```Instances``` | Name: ```webserver-backend``` | VPC: ```Load-Balancer-vpc``` | **Next** karein.
- Dono private servers par tick lagayein, **Include as pending below** karein, aur **Create target group** kar dein.
- **Include as pending below** ka matlab:
  - Jab aap targets select karte hain (jaise aapke EC2 instances), toh AWS unhe direct add nahi karta. Woh pehle unhe ek "Draft" ya "Pending" state mein dalta hai.
  - Include as pending below par click karte hi aapke selected instances niche ek alag table (Review table) mein chale jaate hain.
  - Iska matlab hai: "Maine in instances ko select kar liya hai, aur main confirm karta hoon ki inko target group mein shamil kiya jaye."
 
- **Load Balancers** par jayein -> **Create load balancer** -> Application Load Balancer (Create).
- Name: ```mera-loadbalancer``` | Scheme: ```Internet-facing```.
- **Network mapping**:
  - VPC: ```Load-Balancer-VPC```.
  - Mappings: Yahan dhyan se sirf public subnets chunein:
    - **Zone 1a**: Select karke dropdown se chunein ```public-subnet-1a```.
    - **Zone 1b**: Select karke dropdown se chunein ```public-subnet-1b```.
- Security Group: Default hata kar alb-sg select karein.
- Listeners: HTTP:80 ke aage forward to mein apna Target Group ```webserver-backend``` chunein.
- **Create load balancer** par click kar dein.

<br>

**Network Mapping ka matlab samjhao**:
- Jab hum Application Load Balancer create karte hain to AWS backend mein ALB ke liye node create karta hai, to ye node hamesha public subnet mein rehte hain. Isliye uper hu humne do availability zones select kiye hain.
  - Jab aap ek ALB create karte hain aur use kam se kam do Availability Zones (AZs) assign karte hain, toh AWS har ek AZ mein apne nodes setup karta hai.
    - Yeh nodes asal mein AWS ke backend par chalne wale specialized containers ya optimized virtual instances hote hain.
    - In par sirf AWS ka load balancing software (Hyperplane architecture) chalta hai
   
  - Aapke account mein chalne wale EC2 instances aur ALB nodes mein bada farq hai:
    - Yeh nodes aapke AWS account ke EC2 dashboard par nahi dikhte. Aap inka operating system access (SSH) nahi kar sakte aur na hi inme koi software install kar sakte hain.
    - Inka operating system update karna, security patches lagana, aur scaling karna poori tarah AWS ki zimmedari hoti hai.
    - Aapke liye yeh ek service (Load Balancer) ki tarah kaam karta hai, jisme aapko backend servers ki tension nahi leni hoti.
   
**Yeh Nodes Kaise Kaam Karte Hain?**:

Jab koi user aapki website (http://amazonaws.com) open karta hai, toh yeh process hoti hai:
- DNS Resolution: Route 53 ya DNS server website ke naam ko un ALB nodes ke Public IP addresses mein convert karta hai. (Isiliye ALB ka hamesha ek Static IP nahi hota, balki ek DNS Name hota hai).
- User ka request direct aapke EC2 instance par nahi jaata. Woh pehle us AZ ke ALB Node par pahunchta hai.
- Woh ALB Node request ko check karta hai (Rules aur Path ke hisaab se) aur fir use aapke target group mein maujud actual EC2 instances (jo aapne pichle step mein connect kiye the) par forward kar deta hai.

**Automatic Scaling (Scaling Kaise Hoti Hai?)**:

Agar aapki website par achanak bohot zyada traffic aa jaye, toh AWS backend par in ALB nodes ka size bada kar deta hai ya fir aur naye nodes add kar deta hai (Scale out). Jab traffic kam hota hai, toh nodes apne aap kam ho jaate hain. Aapko iske liye koi Auto Scaling Group setup nahi karna padta.

### Step 9: Final Production Test

Ab hum test karenge ki kya Load Balancer traffic ko distribute kar rha hai ye nhi:
- Load Balancer ka **DNS name** copy karein.
- 2-3 minute wait karein (taaki NAT Gateway ke raste dono servers par Nginx download complete ho jaye aur woh healthy ho jayein).
- Browser ke naye tab mein URL paste karein.
- **Important Note**: ALB ka DNS NAME ke sabse aage by default ```https://``` laga hota hai. Uper lab mein humne ```http``` type ka load balancer banaya hai. To defulat DNA NAME se ```https``` ko hata ke ```http``` kar dein. Tab jake apki website access ho payegi. Agar aap bina http hataye ALB ke DNS NAME ko browser mein access karenge to website access nhi hoti aur error aayega.

Aapko alternate refreshes par **Hello from Asia Webserver** aur **Hello from Europe Webserver** dikhega.

By-default ALB round-robin algorithm use karta hai.

## Request Ka Flow: Step-by-Step (User Se lekar Server Tak)

Maan lijiye ek user ne apne laptop par aapka Load Balancer URL hit kiya: ```http://amazonaws.com```.

Ab dekhiye background mein ek-ek component kaise kaam karta hai:

### Step 1: Request Ka Main Gate Par Aana (Internet Gateway)

User ki request internet se hoti hui aapke VPC ke Internet Gateway (IGW) par aati hai. IGW is request ko check karta hai aur dekhta hai ki yeh traffic aapke VPC ke liye hi hai.

<br>

### Step 2: Public Route Table Ka Rasta Dikhana

Request jaise hi IGW ke andar aati hai, Public Route Table (```public-subnet-rt```) active hoti hai.
- Yeh table kehti hai: "Mere paas rules hain. Jo bhi traffic baahar se aaya hai aur Load Balancer ke paas jana chahta hai, use Public Subnets (```public-subnet-1a/1b```) ki taraf bhej do."

<br>

### Step 3: ALB Security Group Ka Security Check

Traffic ab Public Subnet mein baithe Load Balancer (ALB) ke darwaze par khada hai. Yahan ```alb-sg``` (ALB Security Group) check karta hai:
- "Kya request HTTP (Port 80) se aayi hai?" -> Jawaab mila: "Haan.".
- "Source kya hai?" -> ```0.0.0.0/0``` (Anywhere), yani poori duniya allowed hai.
- ALB Security Group gate khol deta hai aur request ALB ke andar enter kar jati hai.

<br>

### Step 4: ALB Ka Decision (Listener Aur Target Group)

ALB ke andar baitha hai Listener (**HTTP:80**). Woh request ko pakadta hai aur apne Target Group (```backend```) ko dekhta hai.
- ALB dekhta hai ki Target Group mein do servers hain (Server A aur Server B) aur dono Healthy hain.
- ALB yahan Round-Robin algorithm chalata hai. Maano is baar usne chunna Web-Server-A ko, jo ki Private Subnet mein chhupa baitha hai.

<br>

### Step 5: Private Route Table Aur Servers Tak Pahunch

ALB ab request ko Private Subnet ki taraf forward karta hai. Yahan Private Route Table (```private-subnet-rt```) check karti hai ki private subnets ke andar ka network routing sahi hai ya nahi, aur request ko secure network channels se hota hua Web-Server-A tak पहुंचा deti hai.

<br>

### Step 6: Web Server Security Group Ka Strict Check

Request ab Web-Server-A ke hardware level par pahunchti hai. Yahan laga hai ```web-server-sg```. Yeh sabse important check hai jo aapko confuse kar raha tha.
- ```web-server-sg``` ka rule kehta hai: "Main kisi bhi request ko andar nahi aane dunga, chahe woh internet se hi kyun na ho. Main sirf USI request ko allow karunga jo ```alb-sg``` (Load Balancer) se aayi hai."

- Kyunki yeh request sach mein ALB se hi aayi hai, iska source match ho jata hai. Server Security Group lock khol deta hai.

<br>

### Step 7: Nginx Ka Response

Server ke andar chal raha Nginx web server request ko receive karta hai. Woh apna static data uthata hai: <h1>Hello from Asia Webserver</h1> aur use return (vapas) bhej deta hai.

<br>

### Step 8: Return Journey (Response User Tak Kaise Gaya?)

- Server A apna response wapas Load Balancer ko deta hai.
- Load Balancer us response ko lekar Internet Gateway (IGW) ke raste user ke browser tak pahuncha deta hai.
- User ke browser par dikh jata hai: Hello from Asia Webserver.
