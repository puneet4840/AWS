# LAB: Permission Boundary

Chaliye, is **IAM Permission Boundary** Production Lab ko bilkul scratch se, ek-ek click aur ek-ek bariki ke saath detail mein samajhte hain.

<br>

### Phase 1: Lab Ka Setup Aur Kahani (Pre-requisites)

Pehle scenario ko dimaag mein bitha lijiye. Aap ek company mein **Lead Cloud Security** Engineer hain. Aapke under ek junior developer kaam karta hai, jiska naam hai **Bob**.

**Bob ka kaam**: Bob ko Lambda functions aur EC2 servers banane hote hain, jiske liye use naye-naye IAM Roles create karne padte hain.

**Security Risk**: Agar aap Bob ko iam:CreateRole ki permission dete hain, toh Bob ek naya role banakar usme AdministratorAccess dal sakta hai. Fir Bob us naye role ko khud use karke aapke pure AWS account ka admin ban sakta hai. Isko cyber security mein **Privilege Escalation** (apni taqat galat tarike se badhana) kehte hain.

**Solution (Permission Boundary)**: Aap Bob se kahenge, "Tum role banao, mujhe koi dikkat nahi hai. Lekin tum jo bhi naya role banaoge, uspar ek maximum limit (Boundary) lagana mandatory hai.

<br>
<br>

### Phase 2: Create the Permission Boundary

Sabse pehle hum woh boundary policy banayenge jo naye banne wale roles ki taqat ko control karegi.
- Apne main AWS Root/Admin account se login karein.
- Search bar mein **IAM** type karke IAM Dashboard par jayein.
- Left-hand side menu mein **Policies** par click karein.
- Top right mein **Create policy** button par click karein.
- Ab aapko do options dikhenge: Visual aur JSON. Aap **JSON** tab par click karein.
- Wahan pehle se likha hua saala code mita dein aur neeche diya gaya production JSON paste karein:

```
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowCoreServices",
            "Effect": "Allow",
            "Action": [
                "s3:*",
                "ec2:*",
                "cloudwatch:*"
            ],
            "Resource": "*"
        },
        {
            "Sid": "StrictDenyIAMOperations",
            "Effect": "Deny",
            "Action": [
                "iam:DeleteUser",
                "iam:DeleteRole",
                "iam:CreateUser"
            ],
            "Resource": "*"
        }
    ]
}
```

**Iska Matlab Kya Hai?**: Yeh policy kehti hai ki is boundary ke under jo bhi aayega, woh S3, EC2 aur CloudWatch ka poora kaam kar sakta hai (```Allow```). Lekin woh kabhi bhi koi naya IAM User nahi bana sakta, aur na hi kisi User/Role ko delete kar sakta hai (```Deny```).

- Screen ke sabse neeche right side mein **Next** button par click karein.
- Policy name mein dalein: ```Prod-App-Boundary```.
- Description mein likhein: ```Maximum permission safety net for junior developers```.
- Sabse neeche jaakar **Create policy** par click kar dein. Aapki Boundary Policy taiyar hai!

### Phase 3: Create Developer's Permission Policy (With Conditions)

Ab hum Bob (Developer) ke liye ek aisi permission policy banayenge jo use sirf tabhi role banane degi jab woh upar wali boundary ko apply karega.

Ab hum Developer ke liye regular permission policy banayenge jisme hum use naya role banane ki power denge, par ek shart (Condition) ke saath ki use Prod-App-Boundary use karni hi padegi.

- IAM Dashboard ke **Policies** section mein dobara jayein aur **Create policy** par click karein.
- Fir se **JSON** tab par click karein aur purana sample text delete kar dein.
- Neeche diya gaya code copy karke wahan paste karein. **Dhyan dein**: Code ke andar YOUR_ACCOUNT_ID_HERE ko apne asli 12-digit AWS Account ID se replace karein (Aapko aapka Account ID top-right corner mein user name par click karke mil jayega).

```
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "EnforceBoundaryOnRoleCreation",
            "Effect": "Allow",
            "Action": [
                "iam:CreateRole",
                "iam:AttachRolePolicy",
                "iam:PutRolePermissionsBoundary"
            ],
            "Resource": "*",
            "Condition": {
                "StringEquals": {
                    "iam:PermissionsBoundary": "arn:aws:iam::YOUR_ACCOUNT_ID_HERE:policy/Prod-App-Boundary"
                }
            }
        },
        {
            "Sid": "AllowIAMReadOnlyForConsoleView",
            "Effect": "Allow",
            "Action": [
                "iam:Get*",
                "iam:List*",
                "iam:PassRole"
            ],
            "Resource": "*"
        }
    ]
}
```

**Iska Matlab Kya Hai?**:
  - ```iam:CreateRole``` aur ```iam:AttachRolePolicy``` se Bob naya role bana sakega aur usme policies attach kar sakega.
  - Lekin Condition block mein humne likha hai: ```iam:PermissionsBoundary``` equals ```Prod-App-Boundary```. Matlab AWS ka manager tabhi is request ko pass karega jab Bob naye role par yeh specific boundary lagayega.
  - Dusra block (```iam:Get*```, ```iam:List*```) isliye hai taaki Bob AWS Console par roles ki list dekh sake, varna use console par har jagah error dikhega.

- **Next** par click karein.
- Policy Name mein dalein: ```Developer-IAM-Control-Policy```.
- Sabse neeche jaakar **Create policy** par click kar dein.

### Phase 4: Create Junior Developer User (Bob)

Ab hum Bob ka account banayenge aur uspar yeh restrictions lagayenge.
- IAM Dashboard ke left menu mein **Users** par click karein.
- Right side mein **Create user** par click karein.
- User name dalein: ```Junior-Dev-Bob```.
- **Provide user access to the AWS Management Console** wale checkbox par tick lagayein.
- "Are you providing console access to a person?" mein select karein: **I want to create an IAM user**.
- Console password mein **Custom password **select karke ek secure password rakh lein (jaise``` BobSecurePass123!```). Uncheck karein "User must create a new password at next sign-in" taaki hume baar-baar badalna na pade. Next par click karein.
- **Set permissions** page par teesra option chunein: **Attach policies directl**y.
- Search bar mein type karein ```Developer-IAM-Control-Policy``` aur uske saamne waale checkbox par tick laga dein.
- **Next** par click karein aur fir **Create user** par click kar dein. Bob ka account active ho chuka hai!

### Phase 5: Live Production Testing (The Real Verification)

Ab hum Admin account se sign out karenge ya fir ek naya **Incognito Browser Window** kholenge.

- AWS Console Login page par jayein. **IAM** User select karein.
- Apna 12-digit Account ID dalein, fir Username mein ```Junior-Dev-Bob``` aur password dalein jo aapne set kiya tha.
- Ab aap Bob ban chuke hain. Chaliye testing shuru karte hain.

<br>

**Test Case 1: Bob Ki Hacking Koshish (Expectation: FAIL)**:

Bob chahta hai ki ek naya role banaye jo use full Admin power de de, bina kisi boundary ke.
- Bob ke console mein upar search bar mein IAM kholin. Left menu se Roles par jayein.
